# Инструкция по обновлению сайта (Deployment Guide)

Этот файл содержит шаги для ручного обновления сайта на хостинге после внесения изменений в репозиторий GitHub.

## Предварительные требования
1.  У вас настроен SSH-доступ к серверу.
2.  Вы находитесь в консоли сервера.

## Пошаговая инструкция

### 1. Переход в папку проекта
Перейдите в директорию, где размещен ваш сайт. Путь может отличаться в зависимости от настроек хостинга.

```bash
# Пример для cPanel / обычного хостинга
cd ~/www/site_1
# ИЛИ
cd ~/public_html
```

### 2. Получение обновлений (Git Pull)
Загрузите последние изменения из репозитория GitHub.

```bash
git pull origin master
```

> **Примечание:** Если возникают ошибки из-за локальных изменений в конфигурационных файлах, вы можете временно скрыть их:
> ```bash
> git stash
> git pull origin master
> git stash pop
> ```

### 3. Обновление зависимостей (Python Packages)
Если файл `requirements.txt` был изменен, необходимо обновить библиотеки в виртуальном окружении.

```bash
# Активация виртуального окружения (путь может быть venv, env, .venv)
source venv/bin/activate

# Установка/обновление пакетов
pip install -r requirements.txt
```

### 4. Применение изменений базы данных
Проект использует SQLAlchemy.
-   Если добавлены **новые модели**, они создадутся автоматически при перезапуске.
-   Если изменены **существующие поля**, миграции нужно применять вручную или через Flask-Migrate (если настроен).
-   Текущая конфигурация `db.create_all()` только создает отсутствующие таблицы.

### 5. Перезапуск приложения
Чтобы изменения вступили в силу, перезапустите веб-сервер.

**Для Passenger (cPanel, ISPmanager):**
```bash
mkdir -p tmp
touch tmp/restart.txt
```

**Для Systemd (VPS, Gunicorn):**
```bash
sudo systemctl restart myapp
```

## Краткая сводка команд (Cheat Sheet)

```bash
cd ~/www/site_1
git pull origin master
source venv/bin/activate
pip install -r requirements.txt
touch tmp/restart.txt
```
